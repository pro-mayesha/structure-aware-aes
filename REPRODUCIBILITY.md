# Reproducibility Guide

This document is for reviewers and researchers reproducing the **controlled validity study** of document-structure features for holistic essay scoring on PERSUADE 2.0.

## Project purpose

This repository tests whether **discourse-derived document structure**—especially **boundary geometry** and **rhetorical transitions**—adds predictive signal for holistic scores **beyond** essay text, surface features, and component counts.

The primary estimator is **TF-IDF + Ridge** (Steps 1–4). Step 5 is an **optional** DeBERTa robustness check.

**Important framing:** This is an **oracle / upper-bound** analysis using **human discourse annotations**. It is not a deployable production scorer or a neural state-of-the-art benchmark.

## Dataset (not redistributed)

Raw **PERSUADE 2.0** files are **not** included in this repository. Obtain them separately under the dataset license (see [REALISE](https://github.com/scenario-nlp/REALISE)).

### Expected file placement

Place the official CSVs at:

```text
data/persuade20/raw/persuade_corpus_2.0_train.csv
data/persuade20/raw/persuade_corpus_2.0_test.csv
```

See also `data/persuade20/raw/README.md`.

Step 1 (`audit_persuade20.py`) reads these paths and writes processed essay tables to `data/persuade20/processed/` (not committed; regenerated by audit).

## Environment setup

Requires **Python 3.11+**.

```bash
git clone https://github.com/pro-mayesha/structure-aware-aes.git
cd structure-aware-aes

python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

pip install --upgrade pip
pip install -r requirements.txt
```

For **Step 5 (transformer robustness)** only:

```bash
pip install -r requirements-transformer.txt
```

Optional smoke check (no dataset required):

```bash
python scripts/smoke_check.py
```

## Run order

Run from the repository root. Configs live in `configs/`; outputs are written under `outputs/` (precomputed copies are included for inspection without re-running).

| Step | Purpose | Command | Primary output |
|------|---------|---------|----------------|
| 1 | Dataset audit & join validation | `python scripts/audit_persuade20.py` | `outputs/persuade20_audit/` |
| 2 | Feasibility screening (train only) | `python scripts/run_structure_feasibility.py` | `outputs/structure_feasibility/` |
| 3 | Stress tests & ablations | `python scripts/run_structure_stress_tests.py` | `outputs/structure_stress_tests/` |
| 3.5 | Verification & claim refinement | `python scripts/run_structure_verification.py` | `outputs/structure_verification/` |
| **4** | **Final held-out evaluation (paper main result)** | `python scripts/run_final_heldout_evaluation.py` | `outputs/final_heldout_evaluation/` |
| 5 | Optional transformer robustness | `python scripts/run_transformer_robustness.py` | `outputs/transformer_robustness/` |

### Makefile shortcuts

```bash
make setup              # install core dependencies
make smoke              # repo structure + import check (no data)
make audit              # Step 1 (requires raw data)
make reproduce-main     # Step 4 only (requires Steps 1 + processed data)
make reproduce-controls # Steps 2–3.5 (train-only development)
make reproduce-all      # Steps 1–4 sequentially
make reproduce-transformer  # Step 5 (optional; GPU/MPS recommended)
```

### Custom configs

Each runner accepts `--config`:

```bash
python scripts/run_final_heldout_evaluation.py --config configs/final_heldout_evaluation.yaml
```

## What gets regenerated vs shipped

- **Shipped in git:** Scripts, YAML configs, and **frozen experiment outputs** under `outputs/`.
- **Not shipped:** Raw PERSUADE CSVs, `essays.csv` / `essays.parquet` (from audit), DeBERTa embedding cache.
- **Re-running Step 4** overwrites only `outputs/final_heldout_evaluation/` (other step outputs are untouched by design).

## Runtime expectations (approximate)

| Step | Hardware | Time |
|------|----------|------|
| 1 Audit | CPU | ~5–15 min |
| 2–3.5 | CPU | ~30–90 min each |
| 4 Final eval | CPU | ~15–45 min |
| 5 Transformer | CUDA GPU (full fine-tune) or MPS (frozen-encoder fallback) | 1–8+ hours |

Step 5 on Apple MPS uses **frozen DeBERTa embeddings** when CUDA is unavailable; see `outputs/transformer_robustness/compute_report.md`.

## Limitations to disclose when citing results

1. Structural features use **gold human discourse annotations** (oracle structure).
2. Text model is **TF-IDF + Ridge**, not a tuned neural AES system.
3. Official test set is evaluated **once** in Step 4 under a frozen protocol.
4. Step 5 transformer results are **supplementary** and used a frozen encoder on non-CUDA hardware.
